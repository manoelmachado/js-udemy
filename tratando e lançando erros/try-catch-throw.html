Throw, try, catch e finally.

A condição try define uma linha de código para -tentar- ser executada.

A condição catch define uma linha de código para capturar qualquer erro.

A condição finally define uma linha de código para ser executada, independente do resultado.

A condição throw define um erro padrão.

Erros vão acontecer!

Quando estiver executando códigos JavaScript, diferentes tipos de erros podem acontecer.

Esses erros podem ser: erros de código do programador, erros causados por inputs errados e outras coisas imprevistas...

Exemplo: nesse exemplo, nós iremos escrever o "alert" como "adddlert" errado deliberadamente para produzir um erro:
try {
adddlert("Bem-vindo usuário!");
}
catch(err) {
document.getElementById("demo").innerHTML = err.message;
}

O JavaScript pega o "adddlert" como um erro e executa o código do catch para cuidar disso.

JavaScript try e catch

A declaração try permite a você definir um bloco de código para ser testado para erros enquanto ele é executado.

A declaração catch permite a você definir um bloco de código para ser executado, caso aconteça algum erro no bloco de código do try

As declarações try e catch vem em pares:

try {
bloco para ser executado como try
}
catch(err) {
bloco para ser executado para cuidar das mensagens de erro
}

Declaração THROW

A declaração throw te permite a criar um erro padrão.

Tecnicamente você pode "jogar uma exceção" (jogar um erro).

A exceção pode ser uma String, um Number, um Boolean ou um Objeto.

throw "Too big"; // texto

throw 500; // número
